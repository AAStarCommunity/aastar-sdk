# aastar-sdk V5.3 模块开发计划

## Context

SuperPaymaster 合约已升级到 V5.3.0（x402 结算、MicroPaymentChannel、Agent Sponsorship），但 aastar-sdk **完全没有 V5 代码**。ABI 还是 2026-01-25 的 V3 版本，缺少全部 V5.3 函数。

**目标**: 在 aastar-sdk 中新增 3 个 package + 扩展 core，支持 x402 支付、MPC 通道、CLI 工具。然后在 SuperPaymaster 添加 HMAC 中间件。

**约束**: 不修改任何现有 API，所有变更纯粹 additive。


## Step 1: 扩展 @aastar/core — 新 L1 Action 文件

**所有新文件**，不修改 superPaymaster.ts 等现有文件。

### 新建文件

| 文件 | 说明 |
|------|------|
| `packages/core/src/actions/x402.ts` | x402 结算 actions (settle, nonces, fees, earnings) |
| `packages/core/src/actions/agent.ts` | Agent 赞助 actions (isRegistered, rate, policies) |
| `packages/core/src/actions/channel.ts` | MPC 通道 actions (open, settle, close, getChannel) |

### L1 Action 模式 (照搬 superPaymaster.ts)

```typescript
// 双层柯里化: factory(address)(client) => { methods }
export const x402Actions = (address: Address) =>
  (client: PublicClient | WalletClient): X402Actions => ({
    async settleX402Payment({ from, to, asset, amount, ... , account }) {
      try {
        validateAddress(from, 'from');
        return await (client as any).writeContract({
          address, abi: SuperPaymasterABI,
          functionName: 'settleX402Payment',
          args: [...], account: account as any, chain: (client as any).chain
        });
      } catch (error) {
        throw AAStarError.fromViemError(error as Error, 'settleX402Payment');
      }
    },
    // ... 其他方法
  });
```

### x402Actions 方法清单

| 方法 | 类型 | 合约函数 |
|------|------|---------|
| `settleX402Payment` | write | 8 参数 EIP-3009 结算 |
| `settleX402PaymentDirect` | write | 5 参数直接结算 |
| `x402SettlementNonces` | read | nonce 查重 |
| `facilitatorFeeBPS` | read | 默认费率 |
| `facilitatorEarnings` | read | 运营商收益 |
| `operatorFacilitatorFees` | read | 运营商自定义费率 |
| `withdrawFacilitatorEarnings` | write | 提取收益 |
| `setFacilitatorFeeBPS` | write | 设置默认费率 (owner) |
| `setOperatorFacilitatorFee` | write | 设置运营商费率 (owner) |

### agentActions 方法清单

|------|------|---------|

## Step 2: 新建 @aastar/x402 包

**高层 x402 支付客户端**，dApp/Agent 开发者直接导入使用。

### 文件结构

```
packages/x402/
├── package.json          # @aastar/x402, deps: @aastar/core, viem
├── tsconfig.json         # extends ../../tsconfig.build.json
├── src/
│   ├── index.ts          # barrel export
│   ├── X402Client.ts     # 高层客户端 (extends BaseClient)
│   ├── types.ts          # X402PaymentParams, X402Quote, X402Settlement
│   ├── eip3009.ts        # EIP-3009 签名工具 (signTypedData)
│   └── payment-header.ts # X-PAYMENT header 编解码 (base64 JSON)
└── __tests__/
    └── index.test.ts     # smoke test
```

### X402Client 核心方法

| 方法 | 说明 |
|------|------|
| `createPayment(params)` | 生成 nonce + 签 EIP-3009 + 返回 header |
| `settleOnChain(params)` | 调用 L1 settleX402Payment |
| `settleDirectOnChain(params)` | 调用 L1 settleX402PaymentDirect |
| `getQuote()` | 读取费率 |
| `checkNonce(nonce)` | 查重 |
| `x402Fetch(url, init?)` | 自动处理 402: 请求→收到 402→签名→重试 |

### 关键实现: eip3009.ts

```typescript
const EIP3009_TYPES = {

    { name: 'to', type: 'address' },
    { name: 'value', type: 'uint256' },
    { name: 'validAfter', type: 'uint256' },
    { name: 'validBefore', type: 'uint256' },
    { name: 'nonce', type: 'bytes32' },
  ],
} as const;
// 使用 viem signTypedData, domain: { name: "USDC", version: "2", chainId, verifyingContract: usdcAddress }
```

## Step 5: HMAC Challenge 中间件 (SuperPaymaster 仓库)

在 SuperPaymaster 的 `packages/x402-facilitator-node/`（Hono HTTP 后端服务）添加可选 HMAC 中间件。**这不是合约，是 TypeScript 后端代码。**

### 新建文件

| 文件 | 说明 |
|------|------|
| `src/middleware/hmac-challenge.ts` | SHA256 无状态 challenge 生成/验证 |
| `src/routes/settle.ts` | 集成 HMAC 验证 (可选) |

### 设计

- `ENABLE_HMAC_CHALLENGE=true` 环境变量启用
- 402 响应包含 `X-Challenge` header (SHA256 nonce)
- 客户端在 X-PAYMENT 中附带 `HMAC(challenge, payment_data)`
- Facilitator 在 /settle 前验证 HMAC
