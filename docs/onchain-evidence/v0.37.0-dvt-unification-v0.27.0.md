## nodeIds strictly-ascending on the BLS wire (#274)

`sortNodeIdsAscending` sorts nodeIds by 32-byte big-endian value + throws on duplicate, applied at all
three wire sites (`packCumulativeT2/T3Signature`, `packBlsPayload` — the latter drives BLS 0x01 +
WebAuthn cumulative 0x09/0x0a). BLS aggregation is commutative, so reordering does not change the
aggregate signature — it only satisfies the new validator's ordered-signerMask/dedup check. Unit-covered
(`bls-packing.nodeids-sort.test.ts` + parity golden updated to ascending order). airaccount suite 861 passed.
