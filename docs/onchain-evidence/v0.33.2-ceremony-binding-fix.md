# v0.33.2 — Tier-2/3 strict-KMS ceremony binding fix (#259, Codex-caught)

## What 0.33.1 missed
0.33.1 fixed the Tier-1 double-sign, but a Codex review of that fix caught a SEPARATE defect it still
shipped: `prepareTransfer`'s `ownerMessageForStrategy` returned `keccak256(messagePoint)` for Tier-2/3, so
`beginCeremony` bound the strict-KMS challenge to `hashMessage(messagePointHash)`. After #257 + the #258 M1
fix, the ONLY owner signature Tier-2/3 produces is the DVT `ownerAuth` over `userOpHash` (aaSignature +
messagePointSignature are skipped). So the ceremony committed to `messagePointHash` while submit signs
`userOpHash` → strict-KMS challenge mismatch (400). Tier-2/3 B3 would STILL fail even with the Tier-1
double-sign fixed.

## Fix
`ownerMessageForStrategy` binds to `userOpHash` for ALL owner-signed strategies (ECDSA, Tier-1, Tier-2/3) —
every current owner signature signs userOpHash.

## Process note
0.33.1 was published from local without review; this Codex-caught defect is exactly why that was wrong.
0.33.2 goes through PR review before publish.
