# v0.32.2 — complete the #254 guard/tier read fix + regression e2e

## What 0.32.1 missed
0.32.1 fixed the guard-address read (`guard()` instead of the removed `getConfigDescription()`), but the
NEW on-chain regression e2e (added this release) immediately surfaced a SECOND facet of the same class:
`readAccountTierLimits` reads `tier1Limit()`/`tier2Limit()` on the account, but the human-readable
`AIRACCOUNT_ABI` (used by `getAccountContract`) only had them INSIDE the removed `getConfigDescription`
struct — the standalone getters were never declared. So `prepareTransfer` → guard-checker `preCheck` still
reverted at the tier read on v0.22.0 accounts ("Function tier1Limit not found on ABI").

## Fix
- `AIRACCOUNT_ABI`: add `tier1Limit()` / `tier2Limit()` standalone getters (exist on-chain; also in the
  core JSON ABI). Remove the dead `getConfigDescription()` declaration (v0.22.0-absent; no caller after
  #254 — a stale declaration is exactly the #254 trap).

## Regression guard (permanent)
`tests/regression/onchain-evidence/v022-preparetransfer-guard-read-e2e.ts` — drives the REAL guard-checker
`preCheck` (full prepareTransfer tiering read path) against a deployed v0.22.0 account. Added to
RELEASE-CHECKLIST §4.
